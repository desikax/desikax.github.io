API GENERATION FOR GENERIC WEBSERVICE

It is a web service for storing, accessing, modifying, retrieving databases shared by a group of web clients. Java technologies are usually used to build web applications where accessing relational databases are through JDBC API but that requires a database system specific JDBC driver that is pre-installed in the application site. In this model, we have implemented the CRUD operations on the databases through JPA (Java Persistence API) where a generic database can be created, and operations performed on them. This stores metadata of the service to be created and operations are performed through it. APIs are created for the Microservice and presented to the developers developing a frontend. The Service will need to store different data bases and present them to different clients.
